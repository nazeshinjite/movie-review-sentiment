# docs

Project bookkeeping that feeds the graded paper and keeps the collaboration honest.

| File | Purpose |
|---|---|
| [`workload-plan.md`](workload-plan.md) | The authoritative plan: lanes (2/2/2, early + late phase per member), dated calendar, report and presentation split, success criteria. |
| [`contributions.md`](contributions.md) | Running per-member contribution log across code, paper, and video. Source for the required **contribution appendix** and a reference for peer evaluations. |
| [`ai-use-log.md`](ai-use-log.md) | Running AI-use disclosure log (USD policy). Source for the paper's disclosure statement and APA citation of any AI tool. |
| [`decisions.md`](decisions.md) | The shared-foundation decisions (seed, split sizes, preprocessing, schema) and any changes. Changing a shared decision invalidates saved artifacts, so it is logged here. |
| [`judge-dev-log.md`](judge-dev-log.md) | Development log for the LLM-as-judge lane (notebook 05): measurements, decisions, and rationale. Source material for the paper's Methods section and talking points. |
| [`tagging-protocol.md`](tagging-protocol.md) | Pre-registered annotation guideline for categorizing LR-vs-NN disagreements. Becomes a paper appendix. |

Keep these current as you work — reconstructing them at the end is how contribution disputes and disclosure gaps happen.
